Before you start

Prerequisites

Lab 1 requires nothing. The modules in the Foundation track cover everything else you need before you reach Lab 3.

You need
  • Basic comfort using a web browser and terminal
  • Ability to read a log line and follow timestamps
  • The lab platform running at http://localhost
  • About 1 hour of uninterrupted time per lab
You do not need
  • Prior cybersecurity or SOC experience
  • Networking or Linux certifications
  • Programming or scripting knowledge
  • Prior experience with Wazuh, Suricata, or TheHive
Windows / WSL2 users: The platform runs identically on Windows. One limitation — Suricata (the network IDS) is disabled because WSL2 cannot capture raw packets. Labs 5, 6, and 7 all include Windows-specific syslog alternatives so you can complete every objective without Suricata.

Platform

Tools you will use

All tools run locally in Docker. No accounts, no cloud subscriptions, no setup beyond starting the platform.

Wazuh
SIEM / XDR
Collects logs from every container, runs detection rules, and indexes everything in OpenSearch. Your primary investigation interface — used in every lab.
Suricata
Network IDS
Inspects network traffic and fires alerts when it sees suspicious connections, C2 beacons, or known attack signatures. Disabled on Windows/WSL2.
TheHive
Case Management
Where incidents live as Cases. You triage alerts, document findings, run observables through Cortex, and close cases with a verdict — exactly as Tier 1 analysts do.
Shuffle SOAR
Automation
Receives Wazuh webhooks and automatically creates TheHive alerts without human intervention. Lab 3 is built around watching this pipeline fire in real time.
Cortex
Threat Intelligence
Enriches observables — IPs, domains, hashes — by querying threat intelligence feeds. Integrated with TheHive so you can run enrichment directly from a case.
Kali Linux
Attacker Terminal
The attacker machine, accessible from your browser at http://localhost/attacker/. You use it to run Hydra, simulated malware, and other attack tools — then switch to Wazuh to investigate what you just did.
Getting started

How to begin

The platform is already running. Three steps and you are in your first lab.

1
Confirm the platform is up
Open http://localhost — you should see the Wazuh dashboard. Open http://localhost/thehive/ and log in as studentN@thehive.local / FortarkLabs1!. If anything is down, ask your instructor.
2
Read the three Foundation modules
Modules 0, 1, and 2 are short reads (55 minutes total). They cover the concepts Labs 1–7 assume you know. You will answer Q1 of Lab 1 with confidence if you read them first — you will guess if you skip them.
3
Open Lab 1 and follow the pre-flight check
Every lab starts with a pre-flight checklist that verifies your agent is active and your Wazuh filters are correctly set. Do not skip it — a wrong filter is the most common reason alerts seem missing. Confirm all green before you trigger anything.
Work in order. Each lab builds on the vocabulary, tool familiarity, and analytical muscle built in the one before it. Lab 7 expects you to already know how to read a Wazuh alert JSON, correlate timestamps, and write a triage note — all skills from earlier labs. Jumping ahead will make the advanced labs significantly harder than they need to be.
